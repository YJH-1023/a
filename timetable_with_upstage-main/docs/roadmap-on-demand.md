# 사용자 업로드형 로드맵 처리

이 구현은 전체 학과 PDF를 미리 수집하거나 정규화하지 않는다. 사용자가 올린 학과 한 페이지 이미지를 요청 시점에 Gemini로 읽고, 공통 스키마로 검증한 뒤 사용자가 확정한 과목명만 개설과목 목록에서 강조한다.

처리 순서는 다음과 같다.

1. 적용된 입학연도, 학과 코드, 현재 학년, 조회 학기를 요청에 포함한다.
2. Gemini는 선택 학년·학기 칸에 직접 배치된 과목과 이미지상 위치 근거를 반환한다.
3. `validateRoadmapForTarget`이 `exact` 배치이며 선택 학년·학기와 일치하는 항목만 남기고 중복을 제거한다.
4. 사용자가 과목명을 검토·수정하거나 직접 추가한 후 확정한다.
5. 개설과목 목록은 숨기지 않는다. 확정된 로드맵 과목과 정규화된 과목명이 같은 행만 강조한다.

원본에 학기 정보가 없으면 `year_only`, 위치가 불명확하면 `unspecified`로 반환해야 하며 서버가 이를 임의로 `exact`로 바꾸지 않는다. 이 항목들은 선택 학기 자동 강조에서 제외된다. 로드맵은 추천 참고자료이며 졸업요건 충족 판정에는 사용하지 않는다.
